# Transformer架构

由注意力机制为底层，对每个词有Q，K，V三个值，进行计算可以获得词的权重，获取关联

### 自注意力

是指输入同一个序列，通过不同的参数矩阵计算，得到该序列的每个元素对其他元素的注意力分配

### 掩码自注意力

使用掩码遮蔽特定token，让模型只能通过历史信息来预测未来信息，这是Transformer架构进行预测的原因

### 多头注意力

之前的注意力机制都是串行单一的，每次只能计算一种相关关系，所以使用多头注意力，并行多个注意力机制，每次的计算，每个不同的注意力头能够拟合语句中的不同信息，最后拼接起来作为最后输出，即可更全面深入地拟合语言信息

## 天气agent

```python
import os
import re
import requests
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()

client = OpenAI(api_key = os.getenv("API_KEY"),
                base_url="https://api.deepseek.com")

MODEL = "deepseek-chat"

CITY_REGEX = re.compile(
    r'([\u4e00-\u9fa5]{2,8}|[A-Za-z\s]{3,25})', 
    re.IGNORECASE
)

def get_weather(adcode):
    url = "https://restapi.amap.com/v3/weather/weatherInfo?city=" + adcode + "&extensions&output=json&key=" + os.getenv("AMAP_API_KEY")

    response = requests.get(url)
    data = response.json()
    if data["status"] == "1":
        return data["lives"][0]
    else:
        return None
    
def talk_to_model(message):
    response = client.chat.completions.create(
        model=MODEL,
        messages=[{"role": "system", "content": "你是一个查询中国城市天气信息的智能助手。用户会输入一个包含城市名称的句子，你需要从中提取出城市名称，查找对应的adcode，然后调用该文件中的get_weather函数获取天气信息，并以json格式返回给用户。"},
            {"role": "user", "content": message}
        ]
    )
    return response.choices[0].message.content
    

    
def extract_city(text: str):
    """从一句话中提取第一个可能的城市名"""
    match = CITY_REGEX.search(text)
    if match:
        city = match.group(1).strip()
        # 过滤明显不是城市的词
        exclude = {"今天", "明天", "后天", "天气", "气温", "怎么样", "热吗", "冷吗", "下雨", "查查", "帮我", "现在", "多少度"}
        if city not in exclude and len(city) >= 2:
            return city
    return None

print("这是一个查询中国城市天气信息的智能助手。")
user_input = input("user: ")

response = talk_to_model(user_input)

print(response)
```
