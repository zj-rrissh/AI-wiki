# Agent设计思维

一种区别于传统程序设计的范式：强调约束与引导，而非精确控制。

## 定义

Agent 由**决策中心（大模型）**和**工具集**组成，自主决定是否调用工具、如何处理信息。

## 核心对比

| 维度 | 传统程序 | Agent |
|------|----------|-------|
| **控制方式** | 完全掌控 | 约束引导 |
| **决策方式** | 确定性规则 | 概率性推理 |
| **透明度** | 白盒可追溯 | 黑盒不确定 |
| **核心考验** | 架构设计 | 提示工程 |
| **设计哲学** | 精确、透明、可控 | 约束、引导、概率 |

## 关键洞察

### 1. 从「精确可控」到「约束引导」

传统程序：输入 → 固定规则 → 确定性输出
Agent：输入 + 约束 → 大模型推理 → 概率性输出

### 2. Prompt Engineering 的初衷

> "就像在教育一个有着庞大知识储量的新手。虽然他知识储备大，但是不知道怎么开始。这就需要我们进行提示和约束。"

### 3. 设计思维转变

```
准确 → 约束
透明 → 黑盒
可控 → 引导
```

## 应用场景

- **工具调用**：让 Agent 学会使用外部 API
- **RAG**：让 Agent 学会检索知识
- **多 Agent 协作**：让 Agent 分工完成复杂任务

## 前提与局限性

- **概率性输出**：同一输入可能产生不同输出
- **幻觉风险**：模型可能生成错误但看似合理的答案
- **调试困难**：决策过程不可直接观测
