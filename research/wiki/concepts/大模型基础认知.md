# 大模型基础认知

Transformer 架构的核心概念，是理解 Agent 决策机制的基础。

## 核心概念

### 自注意力（Self-Attention）

输入同一序列，通过不同参数矩阵计算每个元素对其他元素的注意力分配。

- **Q（Query）**：当前词的查询向量
- **K（Key）**：用于匹配的关注点
- **V（Value）**：实际的内容向量

注意力分数 = softmax(QK^T / √d) × V

### 掩码自注意力（Masked Self-Attention）

使用掩码遮蔽未来信息，让模型只能通过历史信息预测下一个 token。这是 Transformer 能生成序列的技术原因。

### 多头注意力（Multi-Head Attention）

并行多个注意力头，每个头学习不同的关联模式，最后拼接输出。

- **并行**：多个注意力机制同时计算
- **多样化**：不同头拟合不同语义关系（语法、语义、位置等）
- **拼接**：各头输出 concat 后通过线性层转换

## Transformer 结构

```
Input → Positional Encoding → Encoder（N层多头注意力+前馈）→ Output
                          ↓
                    Decoder（掩码多头注意力+Encoder-Decoder注意力）
```

## 与 Agent 的关系

1. **Transformer 是 Agent 决策中心的技术基础**
2. **注意力机制决定了 Agent 对上下文的「理解」方式**
3. **生成式模型（GPT）是 Transformer 的 Decoder 输出**

## 关键数据点

- **维度**：多头注意力中 d（模型维度）通常 768 或 1024
- **头数**：常用 8 头或 16 头
- **层数**：BERT-base 12 层，GPT-3 96 层

## 前提与局限性

- **计算复杂度**：O(n² × d)，长文本处理成本高
- **位置编码**：需要特殊设计处理序列顺序
- **幻觉来源**：注意力可能聚焦错误上下文
