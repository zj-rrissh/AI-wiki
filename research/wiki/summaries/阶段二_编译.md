# 阶段二 编译摘要

## 核心结论

1. **工具调用框架的核心是 JSON Schema 定义 + 模型自主选择**
   - 工具列表注入 prompt → 模型输出 JSON → 解析执行 → 结果返回
   - 支持本地（Ollama）和云端（DeepSeek）模型切换

2. **Talk 类实现了带人格设置的对话机器人**
   - tiktoken token 计数
   - 支持 system_prompt 动态修改

3. **Function Calling 是 Agent 感知世界的能力基础**

---

## 支撑证据

- **tool_call.py**：get_weather + calculator 两个工具的完整调用链
- **Talk 类**：token 统计、人格切换、对话历史

---

## 质疑

1. **JSON 解析脆弱**：依赖正则 fallback，格式错误时可能截取错误内容
2. **无状态**：每次调用独立，不保留上下文
3. **工具选择无监督**：模型可能选错工具
